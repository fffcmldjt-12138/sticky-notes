# 混合排序、文件夹操作与编辑交互设计

## 混合自由排序

- 根目录和每个文件夹内部都使用统一的有序列表。
- 列表成员可以是笔记、待办或子文件夹，三种类型可以任意混排。
- 每个成员继续使用 `parentFolderId` 和 `order` 表示位置。
- 拖动到成员之间用于调整顺序；拖到文件夹标题用于放入文件夹。
- 文件夹展开后显示一个“移到上一级”投放区，文件夹内成员拖到该区域即可移出。
- 根目录空白区域仍可接收成员，将其移到根目录。
- 服务端以一次事务更新同一父级内所有受影响成员的连续 `order`，避免重复序号。

## 文件夹操作

- 文件夹标题栏支持右键菜单。
- 菜单包含“重命名”和“删除”。
- 删除文件夹时不删除内容：
  - 直接笔记、待办和子文件夹全部移动到被删除文件夹的上一级；
  - 被提升内容按原顺序插入被删除文件夹所在位置；
  - 文件夹本身进入七天回收站。
- 文件夹展开按钮改为无底色的细箭头图标，折叠时向右、展开时向下，使用旋转过渡。

## 时间选择

- 提醒和 DDL 浮层不再依赖系统 `datetime-local` 的小时、分钟控件。
- 使用应用内日期输入、小时滚动列、分钟滚动列。
- 小时为 `00` 到 `23`，分钟以 `5` 分钟为步长，并允许滚轮滚动。
- 时间选择器内部提供“取消”和“确认”；确认只更新浮层草稿。
- 提醒或 DDL 浮层外层继续提供“清除”“取消”“保存”，只有保存才写入任务。

## Markdown 复制

- 笔记编辑器选中内容后按 `Ctrl+C`，同时写入：
  - `text/plain`：选中内容转换出的 Markdown；
  - `text/html`：选中内容的 HTML。
- 纯文本目标粘贴时得到 Markdown，富文本目标仍可使用 HTML。
- 图片保持 `asset://local/...` Markdown 地址。
- 没有选区时保持浏览器默认复制行为。

## 退出异常

- 为退出路径添加可测试的幂等关闭顺序。
- 托盘退出只调用 `app.quit()`，统一由 `before-quit` 完成计时器停止、分离窗口关闭和主窗口销毁。
- `WindowService.quit()` 与 `DetachedWindowService.beginShutdown()` 必须可重复调用。
- 使用打包版循环启动和退出检查 Windows 原生异常；无法稳定复现时保留诊断日志，不宣称已修复。
