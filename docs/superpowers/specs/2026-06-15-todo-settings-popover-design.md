# Todo 设置浮层设计

## 目标

简化待办编辑行。默认界面只呈现任务完成状态、内容、排序、删除和两个设置入口，提醒与 DDL 的详细控件仅在用户主动点击后出现。

## 任务行布局

- 第一行从左到右为：拖动手柄、大号完成勾选框、任务内容输入框、删除按钮。
- 勾选框使用至少 `22px × 22px` 的可见尺寸，并与内容输入框垂直居中。
- 第二行位于内容输入框下方，只显示“提醒”和“DDL”两个按钮。
- 已设置提醒或 DDL 时，对应按钮使用激活样式并显示简短时间摘要。
- 标签继续显示在按钮下方，但没有标签时不占额外高度。

## 提醒浮层

- 点击“提醒”按钮，在按钮附近显示应用内悬浮小窗。
- 浮层包含提醒时间、清除、取消和保存。
- 保存后调用现有任务更新接口，并将 `reminded` 重置为 `false`。
- 清除将 `remindAt` 设为 `null`，同时重置 `reminded`。

## DDL 浮层

- 点击“DDL”按钮，在按钮附近显示应用内悬浮小窗。
- 浮层包含截止时间、提前 3 天、1 天、6 小时多选、自定义提前时间、清除、取消和保存。
- 浮层内部编辑使用临时草稿，只有保存时才更新任务。
- 清除同时清空 `deadlineAt` 和 `deadlineReminders`。

## 浮层规则

- 同一任务同时只显示一个设置浮层。
- 点击另一个入口时切换浮层。
- 点击浮层外部或按 `Escape` 关闭，不保存草稿。
- 浮层锚定在按钮下方，空间不足时限制在编辑器可视区域内。
- 浮层使用高于任务卡片的层级，不改变任务行高度。

## 测试

- 默认不渲染日期输入和提前提醒选项。
- 点击提醒按钮才出现提醒设置。
- 点击 DDL 按钮才出现截止时间和提前提醒设置。
- 保存、清除、取消、外部点击和 `Escape` 行为分别覆盖。
- 勾选框具备独立样式类，防止回退到浏览器默认小尺寸。
